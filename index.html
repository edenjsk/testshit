GitHub Pages로 웹사이트 배포하기

GitHub Pages는 GitHub에서 정적 웹사이트를 무료로 호스팅할 수 있는 서비스입니다. 다음 절차에 따라 간단하게 웹사이트를 배포할 수 있습니다.

1. 새로운 GitHub 리포지토리 생성

먼저, GitHub에 로그인한 후 새로운 리포지토리를 생성합니다. 리포지토리 이름은 원하는 대로 설정할 수 있지만, username.github.io 형식으로 리포지토리를 생성하면 기본 도메인이 됩니다.

2. index.html 파일 추가

리포지토리에 index.html 파일을 생성하고, 그 안에 웹사이트의 내용을 작성합니다. 이 파일은 웹사이트의 홈 페이지로 작동합니다.

3. GitHub Pages 설정

리포지토리 설정(Settings) 탭으로 이동한 다음, "Pages" 섹션을 찾습니다. "Source" 옵션에서 main 브랜치 또는 docs 폴더를 선택하고 "Save"를 클릭합니다.

4. 웹사이트 접속

설정이 완료되면, 잠시 후 https://username.github.io에서 웹사이트를 확인할 수 있습니다. 리포지토리 이름을 다르게 설정한 경우에는 https://username.github.io/repository-name 주소로 접속합니다.
